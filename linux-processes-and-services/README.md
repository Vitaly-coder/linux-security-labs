Процессы, службы Linux и SSH-аутентификация
Цель работы
Изучение процессов и системных служб Linux, а также механизмов SSH-аутентификации с точки зрения информационной безопасности.
________________________________________
Краткая теория
Процесс — выполняемая программа с уникальным PID.
Служба (daemon) — фоновый процесс, управляемый системой (SSH, cron и др.).
Контроль процессов и служб позволяет выявлять аномалии, снижать поверхность атаки и реагировать на инциденты безопасности.
________________________________________
Ход работы
1. Просмотр процессов
ps aux | head
Отображает список активных процессов.
ps aux | grep ssh
Поиск процессов, связанных с SSH.
top
htop
Мониторинг нагрузки и активности в реальном времени.
________________________________________
2. Управление службой SSH
systemctl status ssh
Проверка состояния SSH.
sudo systemctl restart ssh
Перезапуск службы.
sudo systemctl enable ssh
Включение автозапуска SSH (требует контроля с точки зрения ИБ).
________________________________________
3. Управление пакетами
sudo apt update
sudo apt install htop
Обновление системы и установка утилит мониторинга.
________________________________________
4. Работа с SSH-ключами
ls -la ~/.ssh
Проверка каталога SSH.
ssh-keygen -t ed25519
Создание SSH-ключей (ED25519).
cat ~/.ssh/id_ed25519.pub
Просмотр публичного ключа.
head -n 5 ~/.ssh/id_ed25519
Просмотр заголовка приватного ключа (без раскрытия содержимого).
ls -l ~/.ssh/id_ed25519 ~/.ssh/id_ed25519.pub
Проверка прав доступа:
•	приватный ключ — 600
•	публичный ключ — 644
________________________________________
Вывод
Аутентификация по SSH-ключам является более безопасной альтернативой паролям. Контроль процессов, служб и прав доступа является базовым навыком специалиста по информационной безопасности.
